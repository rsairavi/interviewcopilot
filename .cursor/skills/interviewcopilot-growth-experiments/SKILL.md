# Growth Experiments

## Core loop

Design experiments around this loop:

1. user lands
2. signs up
3. asks first interview question
4. hits value moment (high-quality answer)
5. returns for another session
6. upgrades after quota pressure

## Experiment pattern

For each change:

- Hypothesis: what should improve and why
- Change: smallest implementation
- Metric: single success metric
- Guardrail: what must not regress
- Observation window: when to evaluate

## High-leverage areas

- hero CTA clarity and trust proof
- login/signup friction
- session start friction (mic, resume, manual fallback)
- quota messaging and upgrade CTA timing
- post-session retention hooks (history, dashboard insight)
